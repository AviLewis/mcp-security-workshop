# Workspace

A "normal" file in your server's workspace — and a second thing for `list_workspace()`
(the tool you add by hand) to list besides `notes.txt`.

- **Task 1:** the real lesson is in **`notes.txt`** — read THAT through your MCP server
  (`read_workspace_file`) and it explains, step by step, how the read you just performed happened.
- **Task 2:** this is the "normal" file a partner's agent reads first. The lesson lands when, right
  after it, the same agent reads **`../example.env`** — a file it was never meant to see. Same tool, no
  extra permission, no warning. That gap is the boundary Task 3 attacks.
